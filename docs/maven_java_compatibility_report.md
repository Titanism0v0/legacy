# Maven 4.x 与 Java 1.8 兼容性冲突排查报告

## 1. 需求
**目标**：成功启动 Spring Boot 后端项目。
**执行命令**：`mvn spring-boot:run`

## 2. 解决需求（尝试过程）
尝试直接运行启动命令，期望后端服务在本地 8080 端口启动。

## 3. 发现问题
命令执行失败，终端输出错误信息：
```
Error: Apache Maven 4.x requires Java 17 or newer to run.
java version "1.8.0_311"
```
**问题分析**：
- **Maven 版本**：系统安装的 Maven 为 4.x 系列。
- **Java 版本**：当前环境变量 `JAVA_HOME` 指向的是 JDK 1.8.0_311。
- **冲突点**：Maven 4.x 自身的运行环境强制要求 Java 17+，而当前环境仅有 Java 1.8，导致 Maven 进程无法启动。虽然项目本身可能配置为兼容 Java 1.8，但构建工具（Maven）本身的运行门槛未满足。

## 4. 解决方案（可选方案）
为了解决此兼容性问题，提出了以下三种方案：

### 方案 A：升级系统 JDK（推荐）
- **操作**：下载并安装 JDK 17 或 JDK 21，将 `JAVA_HOME` 环境变量指向新安装的 JDK 路径。
- **优点**：能够支持 Maven 4.x 运行，且符合 Java 生态演进趋势。
- **注意**：项目 `pom.xml` 中的编译配置仍可保持 `<source>1.8</source>` 以确保产物兼容性。

### 方案 B：降级 Maven 版本
- **操作**：卸载 Maven 4.x，安装 Maven 3.8.x 或 3.9.x。
- **优点**：Maven 3.x 完全兼容 Java 1.8，无需改动 JDK 环境。

### 方案 C：使用 Maven Wrapper
- **操作**：使用项目自带的 `./mvnw` 脚本启动。
- **现状**：检查发现项目中缺失或无法执行 `mvnw`，此方案不可用。

## 5. 最终解决办法
**当前状态**：
用户选择了 **"修改项目为 Java 1.8"** 的方向，试图通过修改 `pom.xml` 配置来解决问题。
- 我们尝试将 `pom.xml` 中的 `<java.version>` 从 `21` 降级为 `1.8`。
- **结果**：`mvn -v` 依然报错。这证实了问题出在 Maven 启动阶段而非项目编译阶段，修改 `pom.xml` 无效。

**最终建议**：
必须在操作系统层面解决环境不匹配问题。请执行以下任一操作：
1.  **安装 JDK 17+** 并配置环境变量。
2.  **安装 Maven 3.x** 并替换当前的 Maven 4.x。

配置完成后，再次运行 `mvn spring-boot:run` 即可。
