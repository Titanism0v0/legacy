# 云服务器部署指南（Windows + Nginx + Spring Boot + Vue2）

> 目标：把前端 `frontend/dist` 部署到 Nginx，并通过 `/api` 把请求转发到后端 `Spring Boot(8080, context-path=/api)`；同时保证 WebSocket 聊天可用（`/api/ws/chat`）。

## 1. 环境准备（你选的是 Windows + Nginx）

1. 安装软件
- JDK 8（项目 `pom.xml` 指定 `java.version=1.8`）
- MySQL 8（脚本使用 MySQL 8 语法）
- Nginx（用于静态托管与反代）

2. 防火墙/安全组放行端口
- 放行 `80`（或你在 Nginx 上配置的监听端口），用于访问前端
- MySQL：仅需在后端能访问到时开放（通常后端和 MySQL 在同一台服务器则无需对公网开放）
- 后端 `8080`：建议仅允许本机/内网访问（通过 Nginx 转发即可，不必对公网开放）

## 2. 配置后端（必须）

编辑 `[src/main/resources/application.yml](src/main/resources/application.yml)`（我已帮你把数据库/JWT/支付域名改成了环境变量形式，避免硬编码）。

你需要在运行后端 jar 的机器上设置以下环境变量（PowerShell / 系统环境变量均可）：

1. 数据库
- `DB_URL`：例如 `jdbc:mysql://localhost:3306/overseas_purchase?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&createDatabaseIfNotExist=true`
- `DB_USERNAME`：例如 `root`
- `DB_PASSWORD`：你的 MySQL 密码

2. JWT
- `JWT_SECRET`：必须是足够长的密钥（HS512 要求 >= 64 字节）。不要用默认值。

3. 支付二维码中“支付链接”的域名
- `PAYMENT_DOMAIN`：例如 `http://your-public-ip` 或 `http://your-domain.com`
- 说明：当你没有配置收款码图片 `payment.qrcode.qrcode-image-path` 时，后端会生成 `PAYMENT_DOMAIN/payment?orderId=...` 的二维码。

## 3. 初始化数据库（推荐一键脚本）

项目已提供：
- `[scripts/bootstrap_db.sql](scripts/bootstrap_db.sql)`：入口 SQL（`schema + data`）
- `scripts/mysql_rebuild_and_verify.sh`：自动识别 Docker/裸机 MySQL，强制 `utf8mb4` 导入并校验

### 3.1 重建模式（测试环境推荐）
> 会清空并重建 `overseas_purchase`，然后导入 `schema.sql + data.sql`

```bash
bash scripts/mysql_rebuild_and_verify.sh --mode rebuild
```

如果你是 Docker MySQL 且 root 密码未在环境中导出：
```bash
MYSQL_DOCKER_PASSWORD='你的mysql root密码' \
bash scripts/mysql_rebuild_and_verify.sh --mode rebuild
```

### 3.2 仅修复编码模式（不重建）
```bash
bash scripts/mysql_rebuild_and_verify.sh --mode encoding-only
```

### 3.3 可选参数（按需）
- `DB_NAME`（默认 `overseas_purchase`）
- `MYSQL_CONTAINER`（默认 `overseas-mysql`）
- `MYSQL_HOST` `MYSQL_PORT` `MYSQL_USER` `MYSQL_PASSWORD`（裸机 MySQL）
- `BACKEND_CONTAINER`（默认 `overseas-backend`）
- `BACKEND_LOG_PATH`（裸机后端日志路径）

## 4. 部署前端（Vue2）

1. 构建前端
在你本地（和代码一致的环境）：
```bash
cd frontend
npm install
npm run build
```

2. 上传 `frontend/dist/` 到服务器
- 把 `dist/` 拷贝到 Nginx 的站点目录（后续 Nginx 配置里会指向它）

## 5. 部署后端（Spring Boot jar）

1. 构建 jar
```bash
mvn clean package
```

2. 上传 jar 到服务器，并在 jar 运行目录启动
重要：后端文件上传与静态读取是基于 `System.getProperty("user.dir") + /uploads/`。

因此你必须：
- 把 jar 放到某个目录（例如 `D:\apps\purchase-system\`）
- 在该目录下创建 `uploads\` 文件夹（可直接新建）
- 用“该目录为工作目录”方式启动 jar

## 6. Nginx 反向代理配置（重点，含 WebSocket）

1. 准备 Nginx 配置文件
- 参考模板：`[docs/nginx-overseas.conf](docs/nginx-overseas.conf)`

2. 修改模板中的两个地方
- `server_name`：建议改成你的公网 IP 或域名（你当前选择不启用 HTTPS/域名，可先用 IP）
- `root`：改成你服务器上 `frontend/dist` 的真实路径
- `proxy_pass`：默认转发到后端 `127.0.0.1:8080`，前提是后端确实在这端口运行

3. 重新加载 Nginx
- 让新配置生效（命令按你的 Nginx 安装方式执行）

## 7. 部署后的验收

1. 打开页面
- 访问 `http://你的服务器IP/`（或你配置的域名）
- Vue 的路由在 `history` 模式下需要 Nginx 的 `try_files ... /index.html` 回退

2. 验证 REST API
- 浏览器打开页面后，用开发者工具 Network 观察是否请求落在：`/api/...`

3. 验证文件上传回显
- 打开个人中心上传头像
- 页面上的头像 URL 一般会是：`/api/upload/avatar/...`
- 需要确保后端运行目录下 `uploads/avatar/` 写入正常

4. 验证 WebSocket 聊天
- 打开聊天页面
- 浏览器控制台应该能建立连接（后端 WebSocket 在 `/api/ws/chat`）
- 如果聊天连接失败，优先检查 Nginx 的 WebSocket Upgrade/Connection 配置

## 8. 常见坑（按出现概率）

1. 上传图片 404
- 通常是后端没在正确工作目录启动，导致 `./uploads` 不是你以为的目录

2. 所有接口 404/502
- Nginx 的 `/api/` 代理没指到后端 8080，或者后端没启动成功

3. 聊天无法连接
- Nginx WebSocket 反代缺少 `Upgrade/Connection`，或 `proxy_http_version 1.1` 没设置

## 9. 编码与数据缺失专项排查

1. 校验数据库字符集
```sql
SHOW VARIABLES LIKE 'character_set_%';
SHOW VARIABLES LIKE 'collation_%';
```

2. 校验分类与商品数据
```sql
SELECT id, name FROM category ORDER BY id;
SELECT COUNT(*) AS product_count FROM product;
```

3. 查看后端最近错误日志
- Docker 模式：
```bash
docker logs overseas-backend --tail=200
```
- 裸机模式：
```bash
tail -n 200 /你的后端日志路径/backend.log
```
