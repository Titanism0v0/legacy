# 个人收款码使用说明

## 一、准备收款码图片

1. 打开微信，进入"我" -> "服务" -> "收付款" -> "二维码收款"
2. 保存你的收款码图片（截图或保存图片）
3. 将收款码图片命名为 `qrcode.png`
4. 将图片放在以下位置之一：
   - **推荐**：`src/main/resources/static/qrcode.png`（项目资源目录）
   - 或者：`frontend/public/qrcode.png`（前端静态资源目录）

## 二、配置收款人信息

在 `src/main/resources/application.yml` 中配置：

```yaml
payment:
  qrcode:
    receiver-name: 你的名字
    receiver-wechat: 你的微信号
    qrcode-image-path: classpath:static/qrcode.png
    payment-domain: http://localhost:8081
```

## 三、使用流程

1. **用户下单**：用户在商品详情页点击"立即购买"或从购物车结算
2. **生成订单**：系统创建订单，状态为"待付款"
3. **点击支付**：用户在订单页面点击"支付"按钮
4. **显示收款码**：
   - 弹出支付对话框
   - 显示订单号和金额
   - **显示你的个人收款码图片**
   - 显示收款人信息
5. **用户转账**：
   - 用户使用微信扫描收款码
   - 输入订单金额（系统会显示具体金额）
   - 在转账备注中填写订单号（可选）
   - 完成转账
6. **确认支付**：用户点击"我已支付"按钮
7. **订单更新**：系统将订单状态更新为"待发货"

## 四、注意事项

1. **收款码图片**：
   - 建议使用清晰的收款码图片
   - 图片格式：PNG、JPG 都可以
   - 图片大小：建议 300x300 像素以上

2. **金额显示**：
   - 系统会在支付页面和对话框中显示订单金额
   - 用户需要手动输入对应金额进行转账
   - 建议在转账备注中填写订单号，方便核对

3. **支付确认**：
   - 用户转账后需要手动点击"我已支付"按钮
   - 系统不会自动检测到账，需要用户主动确认

4. **安全性**：
   - 这是手动转账方案，适合个人项目或毕业设计
   - 生产环境建议使用正式的支付接口（微信支付、支付宝等）

## 五、文件位置

```
项目根目录/
├── src/main/resources/
│   └── static/
│       └── qrcode.png          # 你的收款码图片（放在这里）
├── frontend/public/
│   └── qrcode.png              # 或者放在这里（前端静态资源）
└── src/main/resources/
    └── application.yml         # 配置文件
```

## 六、测试步骤

1. 将收款码图片放在 `src/main/resources/static/qrcode.png`
2. 重启后端服务
3. 刷新前端页面
4. 创建一个测试订单
5. 点击"支付"按钮
6. 应该能看到你的收款码图片和订单金额
7. 使用微信扫描收款码测试（不需要真的转账）
