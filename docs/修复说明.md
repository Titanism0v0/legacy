# 修复说明

## 问题1：数据库字段不存在

### 错误信息
```
Unknown column 'payment_proof' in 'field list'
```

### 解决方法

**方法1：执行SQL脚本添加字段（推荐）**

在MySQL中执行以下SQL：

```sql
ALTER TABLE `order`
ADD COLUMN `payment_proof` VARCHAR(500) COMMENT '支付凭证（转账截图URL）' AFTER `remark`,
ADD COLUMN `payment_time` DATETIME COMMENT '支付时间' AFTER `payment_proof`;
```

或者直接执行文件：`add_payment_fields.sql`

**方法2：临时方案（如果暂时不想更新数据库）**

代码已经做了兼容处理，即使数据库没有这两个字段也不会报错。但支付凭证功能将无法使用。

执行SQL后，需要修改 `Order.java`，将 `@TableField(exist = false)` 改为 `@TableField(exist = true)` 或直接删除这个注解。

## 问题2：二维码显示不出来

### 可能原因

1. **收款码图片文件不存在**
   - 检查文件路径：`src/main/resources/static/qrcode.png`
   - 确保文件存在且可读

2. **图片格式不支持**
   - 支持格式：PNG、JPG、JPEG
   - 确保文件扩展名正确

3. **图片路径配置错误**
   - 检查 `application.yml` 中的配置：
     ```yaml
     payment:
       qrcode:
         qrcode-image-path: classpath:static/qrcode.png
     ```

### 调试步骤

1. **查看后端控制台日志**：
   - 如果看到 "成功加载个人收款码图片，大小: xxx 字节"，说明加载成功
   - 如果看到 "收款码图片文件不存在"，说明路径不对
   - 如果看到 "加载收款码图片失败"，查看具体错误信息

2. **检查文件位置**：
   ```
   项目根目录/
   └── src/main/resources/
       └── static/
           └── qrcode.png  ← 收款码图片应该在这里
   ```

3. **测试图片加载**：
   - 确保图片文件大小合理（建议 < 1MB）
   - 确保图片格式正确（PNG或JPG）
   - 可以尝试重新保存图片

### 临时解决方案

如果暂时没有收款码图片，系统会生成支付链接二维码（但手机无法访问localhost）。

建议：
1. 准备收款码图片
2. 放在正确位置
3. 重启后端服务
4. 查看控制台日志确认加载成功
