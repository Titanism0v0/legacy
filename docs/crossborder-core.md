## 跨境代购闭环（系统定义）

本系统将“跨境代购”定义为：**受监管的委托代买 + 两段式履约 + 证据链留存 + 合规提示与风控边界**的交易平台。区别于普通电商的一段发货，本系统必须在业务与数据上体现“代购采购/入境/国内派送”的链路与可追溯证据。

---

## 1. 角色（Role）与职责边界

- **消费者（USER）**：浏览/下单/付款；确认税费与禁限售提示；在纠纷时提交证据并发起售后。
- **代购商家（SELLER）**：上架代购商品；在订单中执行“采购→上传凭证→发起跨境段发运→补充清关/入库信息→发起国内段发货”；配合售后举证。
- **平台审核/客服（ADMIN）**：处理违规商品/纠纷仲裁；审核证据链完整性；必要时冻结/下架/处罚（MVP 可先做到“查看+提示+人工处理”）。

说明：本系统采用 `user.role`：`USER / SELLER / ADMIN`。

---

## 2. 两段式履约（Two-leg Fulfillment）

跨境代购的履约被拆分为两段：

- **第一段：海外采购与跨境段（代购段）**
  - 用户下单付款后，卖家执行海外采购。
  - 产出证据：采购凭证（小票/订单截图）、跨境物流单号、必要的入库/验货记录。
- **第二段：国内派送（国内段）**
  - 货物到达国内后，由卖家/仓配发起国内快递发货至用户地址。
  - 产出证据：国内物流单号、签收状态、必要的售后举证。

### 推荐订单状态机（跨境模式）

为最小可落地（MVP）与可答辩展示，订单 `status` 在跨境模式下推荐按以下状态推进（其中带 * 的可先“模拟+声明”）：

- `PENDING_PAYMENT`：待付款
- `PURCHASING`：代购采购中
- `PURCHASED`：已采购（已上传采购凭证）
- `INTL_SHIPPING`：跨境段运输中 *
- `CUSTOMS_CLEARANCE`：清关处理中 *
- `WAREHOUSE_INSPECTION`：入库/验货中 *
- `DOMESTIC_SHIPPING`：国内派送中（已填写国内运单号）
- `COMPLETED`：交易完成
- `CANCELLED`：已取消

兼容普通电商模式（非跨境）：仍可使用原有 `PENDING_SHIPMENT / SHIPPED / COMPLETED`。

---

## 3. 证据链（Evidence Chain）

跨境代购的可解释性来自“证据链”，系统以订单为核心沉淀证据，至少包含：

- **采购凭证**：小票/订单截图（证明已代买、采购价格/时间）。
- **跨境物流信息**：跨境段物流单号与节点截图（MVP 可仅记录单号/图片）。
- **入库/验货记录**：入库照片、验货结果（破损/瑕疵/数量核对）。
- **国内物流信息**：国内快递单号/揽收截图。
- **沟通与仲裁证据**：售后申请中的图片/文字说明、审核意见。

系统落地建议：以 `order_evidence` 表按“订单-证据条目”沉淀，每条证据包含 **类型、URL、上传者角色、时间**。

---

## 4. 规则（Rules）：禁限售与税费

### 4.1 禁限售（MVP）

- **目标**：在上架/下单阶段对明显违规品类做拦截或提示。
- **最小方案**：商品字段标记 + 下单前提示。
  - `product.prohibited_flag`：是否禁限售（0/1）
  - 平台可在管理端或数据层维护“禁限售清单”（MVP 先以人工标记为主）

### 4.2 税费（MVP）

- **目标**：提供“税费预估 + 用户确认声明”，不对接真实海关系统。
- **最小方案**：商品/订单记录税费预估信息并展示。
  - `product.tax_rate`：预估税率（可按类目或固定费率）
  - `order.tax_estimated_amount`：订单预估税费金额（或按商品价 * 税率估算）

---

## 5. 风险与边界声明（答辩口径）

- **不做真实清关对接**：不对接海关接口；清关状态可作为“用户可理解的流程节点”用于展示与追踪。
- **不承诺 100% 杜绝假货**：平台提供“降低风险的机制”（凭证、验货、投诉处置、违规下架），不作绝对担保。
- **跨境退货成本高**：售后优先“协商/部分退款/补偿”，退货退款需满足证据与条件（MVP 可先做到售后单据流转与审核）。
